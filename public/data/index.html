How to read the data

Pay attention to how your software is reading the data. The columns commodity_code and trade_value_usd must be read as character and numeric (i.e. not as integer) respectively. Ignoring this will introduce null values in the data after you read it.

The processed data comes in csv format and its compressed in gz. In R it can be used by using data.table without uncompressing it first, and you can also read it by using Python, Stata or almost any statistical software.

We invite you to read our documentation, but here is an R example function to read compressed data:

  fread2 <- function(file, select = NULL,
                     character = NULL, numeric = NULL) {
    messageline()
    message("function fread2")
    message("file: ", file)

    if(str_sub(file, start = -2) == "gz") {
      d <- fread(
        cmd = paste("zcat", file),
        select = select,
        colClasses = list(
          character = character,
          numeric = numeric
        )
      ) %>%
        as_tibble() %>%
        clean_names()
    } else {
      d <- fread(
        input = file,
        select = select,
        colClasses = list(
          character = character,
          numeric = numeric
        )
      ) %>%
        as_tibble() %>%
        clean_names()
    }

    return(d)
  }

Note for Windows users: In order to read the data without uncompressing it first, you need to install zcat beforehand. The easiest option in our opinion is to do that, and one possibility is to install Chocolatey first and then install the GNU Utilities by running this command from Cmd or Power Shell as administrator:

  choco install unxutils
